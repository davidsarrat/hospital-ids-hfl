Handbook

Deployment handbook: Hierarchical Flower with CIC-IDS2017

A from-scratch guide to understand and run the demo: data, Flower nodes, gateways, orchestration, evaluation, and privacy boundaries.

Fast path

If you only want to inspect the infrastructure, start with episode 00 and open the rendered notebook 04. If you want to reproduce everything, follow the episodes in order.

Episode roadmap

#EpisodeWhat you learn
0000 - Mental model and topologyThe complete architecture before touching commands: hospitals, regions, gateways, and the global hub.
0101 - Setup from scratchHow to prepare Python, dependencies, Kaggle credentials, and the main targets.
0202 - Reproducible data pipelineHow CIC-IDS2017 CSVs become six simulated hospital partitions with a non-IID split.
0303 - Flower Runtime: SuperLink, SuperNode, and SuperExecWhat each Flower Deployment Runtime component does and how it connects in Docker.
0404 - Model and hospital ClientAppHow local parquet is loaded, the MLP is trained, and an update is returned to Flower.
0505 - Regional ServerApp and FedAvgHow the regional server aggregates hospitals and writes regional checkpoints.
0606 - Regional gateway and global aggregationThe key piece: the global layer sees regions as clients, not hospitals.
0707 - Orchestration, Docker, and logsHow federations are actuated from a script and what to inspect when something fails.
0808 - Evaluation, privacy, and next stepsHow the result is measured and which limitations should remain explicit.
