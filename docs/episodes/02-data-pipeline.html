02 - Reproducible data pipeline

How CIC-IDS2017 CSVs become six simulated hospital partitions with a non-IID split.

Cleaning

The cleaning step concatenates the CSVs, normalizes column names, finds the Label column, maps BENIGN to 0 and every attack to 1, drops metadata/leakage columns, and keeps numeric features.

What to notice
  • Lines 60-62 fail early if the raw CSVs are missing. This is better than silently creating empty partitions.
  • Lines 64-71 read every CIC-IDS2017 CSV and concatenate them into one simulation input.
  • Lines 75-83 implement the binary label correction: BENIGN is 0 and every attack label is 1.
  • Lines 84-97 drop metadata/leakage columns and keep numeric features only.
  • Lines 103-105 keep attack_type only for partitioning; training splits later contain numeric features plus label.

Non-IID split with seed 123

Partitioning assigns attack groups to hospitals with biased weights. This makes each hospital see a different distribution, which is the interesting case for FL.

Why this split matters
  • Lines 31-51 compress many CIC-IDS2017 attack strings into stable attack groups.
  • Lines 60-72 combine a Dirichlet draw with each hospital's preferred attack groups. This creates non-IID partitions without hand-picking every row.
  • Lines 112-128 assigns rows group by group and repairs minimum benign/attack coverage where possible.
  • The seed controls the random generator, so the same processed parquet produces the same hospital partitions when SEED=123 is used.

Train-only scaling

The MVP demo fits imputation and standardization from the simulated train splits. It then transforms train, validation, and test for every hospital with those values.

What to notice
  • Lines 167-180 fit imputation and standardization from train splits only.
  • Lines 183-193 apply the stored impute/mean/std values to every split and write a compact training table.
  • This is still a simulation-level scaler. A more privacy-preserving version would aggregate local count/sum/sumsq statistics instead of fitting centrally.
