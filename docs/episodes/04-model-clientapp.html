04 - Model and hospital ClientApp

How local parquet is loaded, the MLP is trained, and an update is returned to Flower.

Dataset and MLP

The main model is intentionally simple so FedAvg remains transparent: a tabular MLP with one logit output and BCEWithLogitsLoss.

Local training

Hospital ClientApp

When the node role is hospital, Flower sends initial arrays. The ClientApp loads those weights, trains on /data/train.parquet, evaluates on validation, and returns arrays plus metrics. The num-examples key enables weighted FedAvg.
