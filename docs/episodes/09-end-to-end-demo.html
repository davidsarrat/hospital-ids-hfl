09 - End-to-end demo outputs

How to run the complete demo and interpret the visible training, evaluation, and prediction artifacts.

One command for the live demo

The easiest way to present the system is make demo. It assumes the dataset has already been downloaded and partitioned. It configures Flower profiles, starts a local Flower Deployment Runtime with real SuperLinks, SuperNodes, and SuperExecs, runs one hierarchical training cycle, evaluates the global checkpoint, produces a small prediction table, executes notebook vignettes, rebuilds this handbook, and prints the global metrics summary.

What to notice
  • Lines 54-58 run the fast local demo, which avoids Docker image rebuilds while still using Flower runtime processes.
  • Lines 60-70 define the optional containerized demo for environments where the Docker images are already available or can be built.
  • Lines 130-166 starts or reuses the runtime, prints the SuperLink/SuperNode commands, and submits the hierarchical Flower runs.
  • Lines 168-215 runs evaluation, prediction, notebook execution, metrics printing, and handbook rebuild in order.
