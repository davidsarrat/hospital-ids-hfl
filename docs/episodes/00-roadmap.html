00 - Mental model and topology

The complete architecture before touching commands: healthcare network sites, regions, gateways, and the global hub.

Demo objective

This project simulates healthcare organizations collaborating to train an intrusion detector on CIC-IDS2017 network-flow telemetry. Each site keeps its local network-flow partition. Regions aggregate site models, and the global layer aggregates regional models.

Not clinical data

The hospital naming convention is an infrastructure scenario. The dataset is not medical: there are no patients, diagnoses, scans, lab values, prescriptions, or clinical outcomes. The task is benign-vs-attack classification for network flows.

Roadmap

  1. Prepare the environment, Kaggle CLI, and dependencies.
  2. Download and clean CIC-IDS2017 without committing raw data.
  3. Create six non-IID partitions with seed 123.
  4. Start SuperLinks, SuperNodes, and SuperExecs with Docker Compose.
  5. Run regional federations with healthcare-network sites as clients.
  6. Run the global federation with regional gateways as clients.
  7. Evaluate the global checkpoint and review privacy boundaries.

The invariant to keep in mind

The architecture is only hierarchical if every layer exposes an aggregate to the layer above it. Healthcare-network sites expose model updates to a regional SuperLink; a region exposes one regional checkpoint through a gateway SuperNode; the global SuperLink never sees site partitions or site SuperNodes directly.

Core rules
  • The region is a Flower federation in its own right, not just a Python function.
  • The global layer is another Flower federation whose clients are gateways.
  • Weighted averaging works because sites and gateways both return num-examples.
  • The shared/ directory is for model/checkpoint artifacts, not raw rows.
