05 - Regional ServerApp and FedAvg

How the regional server aggregates healthcare-network sites and writes regional checkpoints.

Shared entry point

The ServerApp reads level from the run config. This lets the same component serve a regional federation or the global federation.

What to notice
  • Lines 16-25 choose regional or global behavior from context.run_config['level'].
  • Lines 28-41 create initial arrays from the configured input dimension and optionally load a previous checkpoint.
  • This is how a later global round can seed regional site training with the previous global model.

Regional federation

Each region uses FedAvg with weighted_by_key='num-examples'. The clients are the healthcare-network sites connected to the regional SuperLink.

What to notice
  • Lines 64-71 instantiate FedAvg and explicitly set weighted_by_key="num-examples".
  • Lines 73-83 start the Flower strategy on the regional grid. The clients sampled here are site SuperNodes connected to the regional SuperLink.
  • Lines 85-90 capture the final aggregated train/evaluate metrics from Flower's Result object.
  • Lines 92-116 save the aggregated regional checkpoint plus metadata needed by the gateway.
  • Line 102 records the region's training-example total so the global layer can weight this region correctly.
