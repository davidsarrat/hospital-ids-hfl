03 - Flower Runtime: SuperLink, SuperNode, and SuperExec

What each Flower Deployment Runtime component does and how it connects in Docker.

Roles

ComponentIn this demo
SuperLinkThe hub of one federation. There is one per region and one global hub.
SuperNodeA client connected to a SuperLink. It can be a hospital or a gateway.
SuperExec ServerAppThe process that executes hfl_cicids.server_app.
SuperExec ClientAppThe process that executes hfl_cicids.client_app next to a SuperNode.

SuperLink and ServerApp

Hospital SuperNode

The hospital connects to its regional SuperLink, and its ClientApp mounts only its own local folder at /data:ro.

RegionGateway SuperNode

The gateway connects to the global SuperLink and mounts /shared, not /data. This lets it return regional checkpoints without reading raw rows.
