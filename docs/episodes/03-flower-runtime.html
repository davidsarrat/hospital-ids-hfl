03 - Flower Runtime: SuperLink, SuperNode, and SuperExec

What each Flower Deployment Runtime component does and how it connects in Docker.

Roles

ComponentIn this demo
SuperLinkThe hub of one federation. There is one per region and one global hub.
SuperNodeA client connected to a SuperLink. It can be a hospital or a gateway.
SuperExec ServerAppThe process that executes hfl_cicids.server_app.
SuperExec ClientAppThe process that executes hfl_cicids.client_app next to a SuperNode.

SuperLink and ServerApp

What to notice
  • Lines 43-52 create one SuperLink service and expose the Control API port to the host.
  • Lines 56-70 create the ServerApp SuperExec. It connects to the generated SuperLink AppIO API, for example region-eu-superlink:9091.
  • The SuperLink is the coordination hub; the ServerApp SuperExec is the worker process that actually runs the application code submitted by flwr run.

Hospital SuperNode

The hospital connects to its regional SuperLink, and its ClientApp mounts only its own local folder at /data:ro.

What to notice
  • Lines 78-89 start a hospital SuperNode and point it to the regional SuperLink Fleet API.
  • Lines 88-89 pass node config into Flower. The ClientApp later reads role, region, hospital-id, and data-dir from this config.
  • Lines 93-104 start the ClientApp SuperExec and mount only this hospital's partition at /data:ro.
  • Because each hospital service gets a different host folder mounted into /data, the ClientApp code can be shared while the data boundary remains per-hospital.

RegionGateway SuperNode

The gateway connects to the global SuperLink and mounts /shared, not /data. This lets it return regional checkpoints without reading raw rows.

What to notice
  • Lines 113-124 define a gateway SuperNode connected to global-superlink:9092, not to a regional SuperLink.
  • Lines 123-124 mark the node as role="region-gateway", which switches the ClientApp into gateway behavior.
  • Lines 125-140 mount ./shared:/shared only. There is no hospital /data mount for a gateway.
  • This is the privacy boundary that makes the global layer see regional models rather than raw local rows.
